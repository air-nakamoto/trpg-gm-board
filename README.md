# trpg-gm-board

# TRPG GM準備ボード

## 🎯 目的
特定のTRPGシナリオごとに、セッション準備の進捗を全タスク数とチェック数から
自動で％計算し、Discordチャンネルに共有するツール。
準備の過程をみんなに見てもらうことで、「参加したい」と思ってもらえる見せ方をする。

## ✨ コア機能
1. シナリオ名ごとに、カテゴリ付きで準備タスクを登録・チェックできる（管理ビュー）
2. 全タスク数とチェック数から進捗％を自動計算し、進捗バーで表示する
3. シナリオ概要＋全体進捗％を整えた「公開ビュー」としてDiscordチャンネルに投稿できる

- **データ構造**: 本READMEと同じ「シナリオ → カテゴリ → タスク」の3段構造を踏襲

## 📊 データ構造（超重要）
```javascript
// scenarios は配列です（シナリオが複数 → 配列）
// 構造は「シナリオ → カテゴリ → タスク」の3段の入れ子
scenarios = [
  {
    id: 1,
    name: "悪霊の家",
    // 公開ビュー用のシナリオ概要フィールド
    overview: {
      system: "クトゥルフ神話TRPG 第6版",
      tendency: "クローズド",      // シティ/クローズド/謎解き 等
      players: "3〜4人",
      playtime: "4時間前後",
      skills: "目星, 聞き耳"        // 推奨技能
    },
    categories: [
      {
        name: "用意する物",
        tasks: [
          { id: 1, title: "公式シナリオPDF", done: true },
          { id: 2, title: "NPCアイコン", done: false }
        ]
      },
      {
        name: "練習したいこと",
        tasks: [
          { id: 1, title: "しゃべりの練習", done: false }
        ]
      }
    ]
  }
]
```

**AI指示時の鉄則**: 「scenariosは配列です。各シナリオはcategories配列を持ち、各カテゴリはtasks配列を持つ3段構造です」

- 進捗％ = シナリオ内の全タスク数 と done が true の数 から自動計算（例：10個中7個 → 70%）
- カテゴリ単位の進捗（例：用意する物 80% / 練習 0%）も同じ計算で出せる

## 🔧 技術構成
- **フレームワーク**: React + Vite
- **Discord連携**: Discord Webhook（ボットの常時起動は不要）
- **デプロイ**: Vercel
- **理由**: シナリオ→カテゴリ→タスクの3段の入れ子データを扱い、チェックで進捗バーがリアルタイム更新されるため、状態管理が得意なReactが適している。Discord投稿はWebhookを使うことで、サーバー常時起動なしで実現できる。

## 🚫 禁止事項
- ❌ scenarios以外のグローバル状態を作らない（状態はReactのuseStateで管理）
- ❌ データ構造の変更禁止（常に「シナリオ→カテゴリ→タスク」の3段の配列）
- ❌ Discordボット化（discord.js等での常時起動）はPhase 1ではやらない（Webhookのみ）
- ❌ ファイル構成の大幅変更禁止
- ❌ Webhook URL をコードに直書きしない（後述の注意を参照）

## ⚠️ Webhook URL の扱いについて（重要）
Discord Webhook の URL は「そのチャンネルに投稿できる鍵」です。
GitHubに公開リポジトリとして上げると、URLが他人に見られて悪用される恐れがあります。
- コードに直書きせず、環境変数（Vercelの環境変数機能）で管理する
- もしうっかり公開してしまったら、Discordのチャンネル設定からWebhookを削除して作り直す
- ※ 詳しいやり方は、開発環境のAIに「Webhook URLを環境変数で管理したい」と相談してください
